RAG pipeline состоит из следующих шагов: chunking, embeddings, retrieval, rerank и генерация ответа.
Router выбирает один из маршрутов: memory, rag или direct.
Для внешних знаний нужно использовать rag и ограничивать контекст 3-5 чанками.
Для вопросов о переписке нужно использовать memory.
Для простого диалога использовать direct без retrieval.
